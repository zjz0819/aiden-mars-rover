🧠 Key Engineering Concepts
▸
Motor — converts electrical energy into rotational motion. The wheels are outputs of the motor.
▸
Differential drive — two independently controlled motors. Speed difference = turning. Both equal = straight.
▸
Port assignment — SPIKE needs to know which port each motor is on. Wrong port = wrong motor moves.
▸
Cable management — a real engineering discipline. Messy cables catch on wheels and cause bugs.
First Drive Blocks
when Hub button pressed
run motor A for (2) seconds
run motor B for (2) seconds
→ Does it go straight? If not — which motor is the problem?

🧠 The Calibration Mindset
This is Aiden's first experience with physical calibration — a core robotics skill:
Rotations ↔ Distance
1 rotation = wheel circumference
Circumference = π × diameter
SPIKE wheel ≈ 5.6cm diameter → ~17.6cm per rotation

Target: 30cm
30 ÷ 17.6 ≈ 1.7 rotations
(Exact value varies by surface friction — always test!)
Square Path with Loop
repeat (4)
move (1.7) rotations forward
turn right (90) degrees
💬 Parent Script

Square doesn't close
"The rover drifted 5cm by the 4th corner. Is the error in the forward move or the turn? How would you test which one?"

DECISION → if 8 < 15, then…
ACTION → motors stop

🧠 Two Sensors, Two Data Types
▸
Distance sensor — returns a number (cm). Compare with < or >.
▸
Color sensor — returns a color name or number. Compare with = (equals).
▸
Bang-bang control — the simplest line-following algorithm: fully turn left OR fully turn right based on light reading. Crude but effective.

🧠 Priority Logic
When two conditions could be true at once, the rover needs to know which wins:
PRIORITY 1: Distance < 10cm → STOP
PRIORITY 2: Color = red → STOP
PRIORITY 3: Color = green → COLLECT
DEFAULT: Drive forward
The order of if-blocks in code determines which condition is checked first. This is sensor priority — a real robotics design decision.
💬 Parent Script
Before the 60-second run
"In 60 seconds, we don't touch it. What do you predict it will do when it reaches the red zone? What about when it gets close to a wall?"
After the run
"What surprised you? Did it do anything you didn't program? What would you add or change for next time?"
