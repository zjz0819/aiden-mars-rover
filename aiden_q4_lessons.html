"The rover's headlamp brightness depends on how dark the Martian night is. A potentiometer sensor reads ambient light (analog input), and we convert it to control how bright the LED shines. The knob controls the light—just like Mars Rover adjusting its lights!"

⏱ Session Timeline

Analog vs. Digital

0:10–0:20

"LED can't be half-bright with digital. PWM (Pulse-Width Modulation) rapidly flicks the LED on/off so fast it looks smooth."

Wiring & Code
Connect pot to A0. LED to pin 11 (PWM pin). Write code: analogRead() → map() → analogWrite(). Upload and test.
0:45–0:58

0:58–1:05

"How is analog input different from digital? Why do we use map()? Why pin 11 specifically?"
🧠 Key Concepts
▸
analogRead(A0) — reads sensor on pin A0, returns 0–1023
▸
map(value, inMin, inMax, outMin, outMax) — converts one range to another (1023→255)
▸
analogWrite(pin, 0–255) — PWM output; only works on PWM pins (marked ~)
▸
Potentiometer — variable resistor; outputs analog voltage as knob turns
▸
PWM pins — ~3, ~5, ~6, ~9, ~10, ~11 on Uno; enable fading
Code Example: Pot Controls LED Brightness
int pot = A0; int led = 11; void setup() { pinMode(led, OUTPUT); } void loop() { int val = analogRead(pot); int bright = map(val, 0, 1023, 0, 255); analogWrite(led, bright); }

🧠 Key Concepts
▸

▸
INPUT_PULLUP — pinMode mode; button reads HIGH when unpressed, LOW when pressed
▸

▸

▸

▸

▸

▸

distance_sensor.ino
